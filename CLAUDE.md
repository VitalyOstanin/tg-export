# Правила проекта tg-export

## Инструменты

- Использовать `uv` вместо `pip` для всех операций с пакетами (`uv pip install`, `uv venv` и т.д.)
- Запускать тесты через `uv run python -m pytest` (не напрямую через venv)
- Запускать CLI через `uv run tg-export` (не напрямую через venv)
- Проверять логику Telethon по его исходникам: `$HOME/devel/Telethon`
- Использовать Python LSP для навигации по коду (findReferences, goToDefinition и т.д.)

## Деструктивные операции с данными экспорта

- Если пользователь просит удалить/исключить ОДИН конкретный чат, удалять ТОЛЬКО этот чат. Если при поиске найдено несколько похожих результатов — ОБЯЗАТЕЛЬНО переспросить, какие именно удалять. Никогда не удалять больше, чем явно запрошено.

## Безопасность данных

- НИГДЕ в коде, комментариях, коммитах, тестах и любых файлах проекта нельзя использовать названия или ID реальных Telegram аккаунтов, чатов, контактов или пользователей
- В коммитах, логах и примерах использовать только вымышленные данные

## Telegram API и экспорт

- НЕ запускать `uv run tg-export run`, `uv run tg-export list` и любые другие команды, обращающиеся к Telegram API, если в системе уже запущен процесс экспорта. Telegram допускает только одну активную сессию — параллельный вызов приведёт к конфликту.

## Качество кода

- При обнаружении ошибок или предупреждений (Pyright, линтеры, тесты) — исправлять или предлагать исправить ВСЕ найденные проблемы, включая предсуществующие, не связанные с текущими изменениями.

## Ограничения Telegram API

- Telegram API не поддерживает подсчёт сообщений по диапазону дат. `messages.search` с `InputMessagesFilterEmpty` игнорирует `min_date`/`max_date`, `get_messages(limit=0)` всегда возвращает общее количество. При фильтре по датам `messages_total` неизвестен — не показывать прогресс-бар с процентами, показывать только текущий счётчик.
